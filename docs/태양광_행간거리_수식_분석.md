## 📐 1. 행간 거리 계산 (가장 중요)

### 입력값
| 변수 | 의미 | 예시값 |
|---|---|---|
| `solarAngle` (D3) | 태양각 (도) | 26° |
| `moduleLength` (E3) | 모듈 길이 (m) | 2.384m |
| `moduleAngle` (F3) | 모듈 경사각 (도) | 15° |
| `landAngle` (G3) | 토지 경사각 (도) | 3° |

### 중간 계산값 (수직/수평 분해)
```javascript
// 모듈의 수직 길이 (높이 방향)
const verticalLength = Math.sin(moduleAngle * Math.PI / 180) * moduleLength;
// 예: sin(15°) × 2.384 = 0.617m

// 모듈의 수평 거리 (지면 투영)
const horizontalLength = Math.cos(moduleAngle * Math.PI / 180) * moduleLength;
// 예: cos(15°) × 2.384 = 2.303m

// 토지 경사 보정 수직길이
const correctedVertical = Math.sin(landAngle * Math.PI / 180) * moduleLength;
// 예: sin(3°) × 2.384 = 0.125m
```

### 핵심 수식 — 행간 거리 (모듈 뒤쪽)
```javascript
// 보정 태양각 거리 = tan(90° - 태양각) × (모듈 수직높이 - 토지경사 보정)
const correctedSolarDistance = 
  Math.tan((90 - solarAngle) * Math.PI / 180) * (verticalLength - correctedVertical);
// 예: tan(64°) × (0.617 - 0.125) = 1.009m

// 행간 거리 (모듈 뒤끝과 다음 모듈 앞끝 사이)
const rowSpacing = horizontalLength + correctedSolarDistance;
// 예: 2.303 + 1.009 = 3.312m
```

### 모듈과 모듈 사이 (실제 빈 공간)
```javascript
// 행간 거리에서 모듈 수평길이를 뺀 것
const moduleToModuleGap = rowSpacing - horizontalLength;
// 예: 3.312 - 2.303 = 1.009m
```

### 적정 거리 (현장 적용)
```javascript
// 사용자가 추가 안전 마진(현장 적용값)을 더할 수 있음
const fieldMargin = 1.0; // m (사용자 입력)
const optimalSpacing = rowSpacing + fieldMargin;
const optimalGap = moduleToModuleGap + fieldMargin;
```

## 📐 2. 경사각 계산

### 수직/수평 거리로 경사각 구하기
```javascript
// 입력: 수직거리 30, 수평거리 73
const verticalDist = 30;
const horizontalDist = 73;

// atan으로 라디안 → 도 변환
const slopeAngleRad = Math.atan(verticalDist / horizontalDist);
const slopeAngleDeg = slopeAngleRad * 360 / (2 * Math.PI);
// 예: atan(30/73) = 22.34°
```

### 기울기 % → 경사각
```javascript
// 입력: 기울기 7%
const slopePercent = 7;
const slopeRatio = slopePercent / 100; // 0.07
const slopeAngleRad2 = Math.atan(slopeRatio);
const slopeAngleDeg2 = slopeAngleRad2 * 360 / (2 * Math.PI);
// 예: atan(0.07) = 4.00°
```

## 📐 3. 수평 투영 면적 (모듈 경사각에 따른 실제 점유 면적)

```javascript
const moduleAngle = 18;       // 도
const moduleLength = 2.384;   // m (경사 길이)
const moduleWidth = 1.303;    // m (모듈 폭)
const moduleCount = 50;       // 장

// 수평 투영 길이
const projectedLength = Math.cos(moduleAngle * Math.PI / 180) * moduleLength;
// 예: cos(18°) × 2.384 = 2.267m

// 수직 투영 (그림자 늘어나는 양)
const verticalProjection = moduleLength - projectedLength;
// 예: 2.384 - 2.267 = 0.117m

// 실제 수평 투영 면적
const projectedArea = projectedLength * moduleWidth * moduleCount;
// 예: 2.267 × 1.303 × 50 = 147.72 m²

// 모듈 자체 면적 (참고용)
const totalArea = moduleLength * moduleWidth * moduleCount;
// 예: 2.384 × 1.303 × 50 = 155.32 m²

// 축소율 (경사각으로 인한 실제 점유 감소)
const reductionRate = (totalArea - projectedArea) / totalArea * 100;
// 예: 4.89%
```

## 📐 4. 모듈 앞면(남측) 지면고

```javascript
const moduleAngle = 23;        // 도
const moduleLength = 2.465;    // m
const centerHeight = 1.1;      // m (중심 높이, 외기둥 기준)

// 경사고 (모듈의 수직 높이)
const slopeHeight = Math.sin(moduleAngle * Math.PI / 180) * moduleLength;
// 예: sin(23°) × 2.465 = 0.963m

// 앞면(남측) 지면고 = 중심 높이 - 경사고/2
const frontHeight = centerHeight - (slopeHeight / 2);
// 예: 1.1 - 0.963/2 = 0.618m
```

## 📐 5. 양각 계산 (밑변 + 양각으로 수직거리 구하기)

```javascript
// 두 각도를 이용한 수직거리 계산 (삼각측량)
const totalDistance = 2.74;  // m (총 수평거리)
const angle1 = 10;           // 도 (토지 경사)
const angle2 = 20;           // 도 (태양각)

const a1Rad = angle1 * Math.PI / 180;
const a2Rad = angle2 * Math.PI / 180;

// 수직 거리 비율
const ratio = (Math.tan(a1Rad) * Math.tan(a2Rad)) / 
              (Math.tan(a1Rad) + Math.tan(a2Rad));
// 예: (0.176 × 0.364) / (0.176 + 0.364) = 0.119

// 수직 거리
const verticalDistance = ratio * totalDistance;
// 예: 0.119 × 2.74 = 0.326m

// 수평거리 변환 (다른 각도 기준)
const targetAngle = 20;
const targetAngleRad = targetAngleDeg * Math.PI / 180;
const horizontalDistance = verticalDistance / Math.tan(targetAngleRad);
```

## 🎯 SolarAdvisor 적용 우선순위

### 🥇 최우선 적용 — 행간 거리 자동 계산
**현재 SolarAdvisor의 한계:**
- 사용자가 행간 거리를 1.2m 등 임의 입력
- 태양각·모듈경사각·토지경사각이 반영 안 됨
- 그늘 침범 가능성

**개선 방향:**
- 위치(위경도) → 동지 태양고도 자동 계산 (예: 진주 26°)
- 모듈 경사각 + 토지 경사각 입력
- 위 §1 수식으로 **이론 행간거리 자동 산출**
- 사용자에게 "권장 행간 X.XXm" 표시 + 안전 마진 추가

### 🥈 2순위 — 수평 투영 면적 보정
**효과:**
- 부지 면적 대비 실제 패널 점유 면적 정확히 계산
- 경사 30° 패널은 13.4% 적게 점유
- 더 많은 패널 배치 가능

### 🥉 3순위 — 토지 경사도 측정 + 보정
**현재:** 평지 가정  
**개선:** 토지 경사 입력 → 행간거리 동적 보정

## 💡 핵심 인사이트

엑셀 파일에서 발견한 실무 노하우:

1. **태양각**은 **동지 태양고도**를 기준으로 설계 (그늘 최악 시점)
   - 한국: 위도 33~38° 기준 → 동지 태양고도 약 26~31°

2. **모듈 경사각**과 **토지 경사각**은 별개로 보정 필요
   - 경사지에서는 행간거리가 평지보다 더 짧아야 함

3. **행간 거리** ≠ **모듈 사이 거리**
   - 행간 거리 = 모듈 앞끝 ~ 다음 모듈 앞끝
   - 모듈 사이 = 모듈 뒤끝 ~ 다음 모듈 앞끝
   - SolarAdvisor UI에서 이 차이를 명확히 구분 필요

4. **경사고/2**로 앞면고 계산
   - 모듈 중심을 기준으로 경사 양쪽으로 ±0.5×경사고
   - 외기둥 구조물 설계 시 필수

5. **현장 적용 마진** (사용자 입력 가능)
   - 이론값 + 안전 마진 (보통 1m)
   - 시공 오차, 청소 통로 확보 등
